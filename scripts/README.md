# Cloudflare Pages Deployment Scripts

هذا المجلد يحتوي على سكريبتات النشر التلقائي على Cloudflare Pages لمشروع open-lovable4.

## الملفات المتوفرة

### 1. `deploy-to-cloudflare.sh` (مفصل)
سكريبت شامل مع معالجة الأخطاء والتحقق من المتطلبات:

**المميزات:**
- ✅ التحقق من وجود جميع المتطلبات (Git, Node.js, npm, Wrangler, GitHub CLI)
- ✅ تثبيت تلقائي للأدوات المفقودة
- ✅ رسائل ملونة ومنظمة
- ✅ معالجة شاملة للأخطاء
- ✅ التحقق من حالة تسجيل الدخول
- ✅ دعم متعدد لأنظمة التشغيل

### 2. `simple-deploy.sh` (بسيط)
سكريبت مبسط مطابق للطلب الأصلي:

**المميزات:**
- ✅ سهل الاستخدام
- ✅ خطوات واضحة ومباشرة
- ✅ مناسب للمشاريع البسيطة

## المتطلبات الأساسية

قبل تشغيل السكريبتات، تأكد من تثبيت:

1. **Git** - لإدارة الكود
2. **Node.js** - لتشغيل المشروع
3. **npm** - لإدارة التبعيات
4. **Wrangler CLI** - لنشر المشروع على Cloudflare
5. **GitHub CLI** - للتفاعل مع GitHub

## كيفية الاستخدام

### تشغيل السكريبت المفصل:
```bash
./scripts/deploy-to-cloudflare.sh
```

### تشغيل السكريبت البسيط:
```bash
./scripts/simple-deploy.sh
```

## الخطوات التي يقوم بها السكريبت

1. **🔑 تسجيل الدخول إلى GitHub** - التحقق من حالة تسجيل الدخول
2. **📂 استنساخ المستودع** - استنساخ أو تحديث المشروع
3. **📦 تثبيت التبعيات** - تثبيت جميع حزم npm
4. **🏗️ بناء المشروع** - تشغيل `npm run build`
5. **🔧 إعداد متغيرات البيئة** - إعداد المتغيرات المطلوبة
6. **🌐 نشر المشروع** - رفع المشروع على Cloudflare Pages
7. **✅ التحقق من النشر** - عرض حالة آخر نشر

## إعداد Cloudflare Pages

### 1. إنشاء مشروع Cloudflare Pages:
```bash
wrangler pages project create open-lovable4
```

### 2. تسجيل الدخول إلى Cloudflare:
```bash
wrangler login
```

### 3. إعداد متغيرات البيئة (اختياري):
```bash
wrangler secret put VARIABLE_NAME
```

## استكشاف الأخطاء

### مشاكل شائعة:

1. **خطأ في تسجيل الدخول إلى GitHub:**
   ```bash
   gh auth login
   ```

2. **خطأ في تسجيل الدخول إلى Cloudflare:**
   ```bash
   wrangler login
   ```

3. **خطأ في بناء المشروع:**
   - تحقق من وجود `package.json`
   - تحقق من وجود script `build`
   - تحقق من تثبيت جميع التبعيات

4. **خطأ في النشر:**
   - تحقق من وجود مجلد البناء (`out`, `.next`, أو `dist`)
   - تحقق من صحة اسم المشروع
   - تحقق من صلاحيات الوصول إلى Cloudflare

## ملاحظات مهمة

- تأكد من أن المشروع متوافق مع Cloudflare Pages
- تأكد من وجود script `build` في `package.json`
- تأكد من أن مجلد البناء يحتوي على الملفات المطلوبة
- احتفظ بمفاتيح API والرموز المميزة في مكان آمن

## الدعم

إذا واجهت أي مشاكل، يرجى:
1. التحقق من رسائل الخطأ
2. التأكد من تثبيت جميع المتطلبات
3. التحقق من صلاحيات الوصول
4. مراجعة وثائق Cloudflare Pages
